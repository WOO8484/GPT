# KNOWN LIMITATIONS (현재 설계상 제한사항)

이 문서는 GPT 공작소의 현재 버전(0.0.5, Phase E)에서 **의도적으로 구현하지 않았거나
범위를 제한한 사항**을 정리합니다. 아래 항목은 버그가 아니라 현재 설계상 의도된 제한입니다.

## Blogger 연동 관련
- **브라우저 직접 호출 시 Blogger API CORS 차단 가능**: 서버/프록시 없이 브라우저에서 Blogger API v3를 직접 호출하므로, 환경에 따라 CORS 정책으로 요청이 차단될 수 있습니다.
- **OAuth 전체 로그인 플로우 미구현**: Access Token은 사용자가 직접 발급받아 수동으로 입력하는 구조만 지원합니다. 앱 내 로그인/토큰 발급 화면은 없습니다.
- **Client Secret/Refresh Token 저장 금지**: 보안상 Client Secret과 Refresh Token은 코드나 저장소 어디에도 저장하지 않습니다. Access Token은 세션(브라우저 메모리) 동안만 유지되며 새로고침 시 다시 입력해야 합니다.
- **Blogger 이미지 업로드/API 변환 미구현**: 이미지를 Blogger 서버나 Google Drive에 업로드하는 기능은 없습니다. dataUrl 이미지는 사용자가 별도로 호스팅 URL로 변환해야 합니다.
- **실제 발행 결과와 로컬 저장 상태가 불일치할 수 있음**: Blogger 업로드 자체는 성공했지만 로컬 저장이 실패하는 경우, 자료실에 표시되는 상태와 실제 Blogger 상태가 다를 수 있습니다. 이 경우 오류 이력과 해당 글의 상태 표시로 확인할 수 있습니다.

## 예약발행 관련
- **자동 예약발행은 서버/Worker 단계에서 구현 예정**: 현재 예약발행은 "예약 정보 저장/관리" 수준까지만 지원합니다. 브라우저가 꺼져 있어도 지정한 시간에 자동으로 발행되는 기능은 없으며, 추후 서버/Worker 연동 단계에서 별도로 개발할 예정입니다.

## 업로드/자료 관련
- **ZIP 업로드는 보류 상태**: metadata.json/content.html/content.md/content.txt 개별 파일 업로드만 지원하며, ZIP 파일을 통째로 업로드/파싱하는 기능은 없습니다.
- **JSZip/vendor 미사용**: 외부 라이브러리(JSZip 등)나 vendor 폴더를 추가하지 않았습니다. 모든 기능은 순수 브라우저 API로 구현되어 있습니다.

## 통계 관련
- **통계는 로컬 저장 데이터 기준**: 통계 화면의 모든 수치는 현재 브라우저(기기)에 저장된 데이터만을 기준으로 계산됩니다. 다른 기기나 외부 서비스의 데이터는 반영되지 않습니다.

## 기타
- 통계·전체 점검 화면은 읽기 전용이며, 이 화면에서 글 데이터를 직접 수정할 수 없습니다.
- 전체 점검은 실제 외부 API를 호출하지 않으며, 현재 저장된 데이터와 모듈 상태만 확인합니다.
