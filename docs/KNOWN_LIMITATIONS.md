# KNOWN LIMITATIONS (현재 설계상 제한사항)

이 문서는 GPT 공작소의 현재 버전(0.0.6 repair2)에서 **의도적으로 구현하지 않았거나
범위를 제한한 사항**을 정리합니다. 아래 항목은 버그가 아니라 현재 설계상 의도된 제한입니다.

## 로그인/세션 관련 (repair1 신규)
- **세션은 브라우저 세션 동안만 유지**: 로그인 성공 시 발급되는 세션 토큰은 sessionStorage에만 저장됩니다. 탭/브라우저를 완전히 종료하면 세션이 사라지며 다시 로그인해야 합니다.
- **세션 만료 확인은 API 호출 시점에만 이루어짐**: 별도의 "세션 유효성 확인" API를 주기적으로 호출하지 않습니다. Blogger 상태 확인/임시저장 등 실제 Worker 호출에서 401을 받았을 때만 로그인 화면으로 돌아갑니다.
- **로그인 시도 횟수 제한 없음**: 브라우저 단에서 별도의 잠금/횟수 제한은 두지 않았습니다. 관련 정책이 필요하면 Worker(서버) 쪽에서 처리해야 합니다.

## Blogger 연동 관련 (repair1에서 Worker 기반으로 변경)
- **실제 발행(즉시 공개) 기능 없음**: 이번 repair1의 Blogger 기능은 "블로그 임시 저장"과 "예약 저장"(로컬 예약 정보)만 제공합니다. Worker에 실제 발행 경로가 없어 즉시 공개 기능은 만들지 않았습니다.
- **자동 예약발행 미구현**: "예약 저장"은 예약 정보를 로컬에 저장하는 수준까지만 지원합니다. 지정한 시간에 브라우저가 꺼져 있어도 자동으로 임시저장/발행되는 기능은 없습니다.
- **Blog ID/Client Secret/Refresh Token은 Worker(서버)가 관리**: 브라우저 코드에는 어떤 형태로도 저장하지 않습니다. 기존처럼 사용자가 Blog ID/Access Token을 직접 입력하는 화면은 더 이상 제공하지 않습니다.
- **dataUrl 이미지의 Blogger 표시 문제는 미해결**: ZIP/수동 업로드로 저장된 이미지는 여전히 dataUrl(브라우저 로컬) 형태입니다. Blogger 임시저장 시 본문에 이미지가 자동으로 올바르게 표시되려면 별도의 이미지 호스팅 URL 변환이 필요하며, 이번 repair1에서는 경고만 표시하고 변환 기능은 제공하지 않습니다.
- **실제 업로드 결과와 로컬 저장 상태가 불일치할 수 있음**: Blogger 임시저장 자체는 성공했지만 로컬 저장이 실패하는 경우, 자료실에 표시되는 상태와 실제 Blogger 상태가 다를 수 있습니다. 이 경우 오류 이력과 해당 글의 상태 표시로 확인할 수 있습니다.

## 업로드/자료 관련
- **ZIP 자동 업로드는 GPT 공작소 v3 자료 패키지 구조 기준**: metadata.json/content.html/content.md/content.txt와 thumbnail.*/body-01~03.* 이미지 파일명 규칙을 기준으로 인식합니다. 이 구조를 벗어난 파일명은 인식되지 않을 수 있습니다.
- **등록하기 저장 조건은 파일/이미지 정상 + SEO 판정 "통과"**: [검증]은 필수 파일 8종(메타데이터/HTML/MD/TXT/썸네일/본문이미지 3장) 구조와 SEO 점수(기존 seo-module.js 80점 이상 통과 기준)를 함께 확인합니다. 구조가 정상이어도 SEO가 "통과"가 아니면 저장할 수 없으며, 미통과 사유가 안내됩니다.
- **등록 후 SEO 재검수 화면 없음**: 이번 대시보드/팝업 구조에는 저장된 글의 SEO를 다시 검수하는 별도 화면이 없습니다. SEO 결과를 개선하려면 ZIP을 보완해 다시 등록해야 합니다.
- **ZIP 안의 이미지가 Blogger 서버 이미지로 자동 업로드되는 것은 아닙니다**: ZIP 이미지도 기존 이미지 관리 기능과 동일하게 dataUrl 방식으로 로컬 저장됩니다.
- **ZIP 읽기는 직접 구현한 최소 리더 사용**: 외부 네트워크에서 JSZip 라이브러리를 내려받을 수 없는 샌드박스 환경 제약으로, `js/vendor/zip-reader.js`(이전 작업에서 직접 작성한 최소 ZIP/DEFLATE 구현)를 사용합니다. 저장(Stored)/Deflate 압축만 지원하며, 암호화된 ZIP·ZIP64(초대형 ZIP)·그 외 압축 방식(BZIP2/LZMA 등)은 지원하지 않습니다.
- **기존 개별 파일 업로드는 보조 기능으로 유지**: metadata.json/content.html/content.md/content.txt를 각각 선택해 업로드하는 기존 방식도 등록하기 팝업 안의 "수동 업로드" 영역에서 계속 사용할 수 있습니다. 이 경로로 저장된 글은 검증 게이트를 거치지 않습니다(기존 0.0.6 동작과 동일).

## 이미지 관리 화면 관련 (repair1 변경)
- **이미지 관리 화면은 이번 대시보드 구조에 연결하지 않음**: js/image-module.js 파일과 기능 자체는 삭제하지 않았지만, 새 단일 대시보드/팝업 구조(등록하기/자료실/블로그 등록하기/설정)에는 이 화면으로 가는 진입점이 없습니다. 자료실 팝업에서 글 선택 시 제공되는 기능은 [미리 보기]/[삭제 하기]뿐입니다.

## 통계 관련
- **통계 화면 진입점 없음**: 이번 대시보드/팝업 구조에는 별도의 "통계 보기" 화면이 없습니다. js/statistics-module.js 자체는 남아 있으며, 설정 팝업의 "전체 점검" 기능에서 관련 로직 일부를 계속 사용합니다.

## 오류 이력 관련
- **오류 백과사전은 기존 단순 이력 구조 유지**: 설정 팝업의 "오류 확인"은 발생 시각/모듈/메시지 위주의 기존 오류 이력 목록을 그대로 사용합니다. 오류번호/버전/해결방법/재발방지 등을 항목별로 구조화한 신규 포맷으로의 전환은 이번 repair1 범위에 포함하지 않았습니다.

## 기타
- 전체 점검은 실제 외부 API를 호출하지 않으며, 현재 저장된 데이터와 모듈 상태만 확인합니다.
- 데이터 초기화는 저장된 글만 초기화하며, 오류 이력은 초기화 대상에 포함하지 않습니다.

## 팝업/화면 흐름 관련 (repair2 신규)
- **스크롤 허용 영역은 오류백과/전체점검/미리보기/자료실(목록)로 한정**: 그 외 팝업(등록하기, 블로그 등록하기, 설정 기본 화면, 검증 결과 화면)은 스크롤이 생기지 않도록 요약형으로 구성했습니다. 내용이 화면보다 길어질 수 있는 입력값(예: 매우 긴 SEO 미통과 사유 목록)은 줄바꿈으로만 표시되며 별도로 축약하지 않습니다.
- **재검증은 동일 파일 재실행**: [재검증] 버튼은 같은 ZIP 파일로 검증을 다시 실행할 뿐, 파일 선택 창을 다시 열지 않습니다. 다른 ZIP으로 바꾸려면 [ZIP 파일 선택]을 다시 눌러야 합니다.

## 미리보기 Blogspot 근사치 관련 (repair2 신규)
- **실제 Blogger 테마와 100% 동일하지 않음**: 미리보기는 content.html을 기준으로 요약 박스/목차/광고 자리/FAQ/관련 글 박스를 갖춘 형태로 근사하게 보여주지만, 실제 Blogger에 적용된 테마 CSS의 영향은 반영하지 않습니다.
- **광고 자리는 고정 위치 표시용 박스**: 본문 시작 전/본문 끝, 2곳에 표시 전용 박스만 제공하며 실제 광고 코드는 삽입하지 않습니다.
- **관련 글은 단순 최근 글 목록**: 관련도 계산 없이 현재 글을 제외한 저장된 글 중 최대 3건을 보여주는 수준입니다.
- **모바일/PC 보기는 폭 전환만 제공**: 실제 기기별 렌더링 차이(폰트, 여백 등)까지 재현하지는 않습니다. 네이버 미리보기는 이번 범위에 포함하지 않았습니다.

## 블로그 지시서 관리 관련 (repair2 신규)
- **OpenAI API 자동 실행 아님**: [GPT 지시서 복사]는 저장된 지시서 텍스트를 클립보드에 복사하는 기능일 뿐, GPT/OpenAI API를 자동으로 호출해 글을 생성하지 않습니다.
- **지시서는 localStorage에 저장**: 브라우저(기기)별로 별도 저장되며, 다른 기기나 브라우저와 동기화되지 않습니다.
- **지원 파일 형식은 .md/.txt만**: 그 외 형식의 파일을 업로드하면 텍스트로 그대로 읽어 저장을 시도하며, 별도의 형식 검증은 하지 않습니다.
