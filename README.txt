GPT 공작소 v1.2
====================

v1.2 변경 내용 (v1.1 Lite 대비)
- 이름 변경: 화면 표시명에서 'Lite'를 제거했습니다.
  (GPT 공작소 Lite → GPT 공작소 / GPT공작소 Lite → GPT공작소)
- 자료실 → 게시판: 화면 표시명을 '게시판'으로 바꿨습니다.
  - 메인 화면에는 최근 글 목록이 게시판으로 표시되며, 최신 업데이트 글이
    먼저 보이도록 최신순으로 정렬됩니다(정렬/저장 구조 자체는 v1.1과 동일).
  - 게시판 패널 헤더의 [전체 게시판] 버튼을 누르면 전체 글 목록 팝업이
    뜨고, 여기서도 최신순으로 표시됩니다.
  - 게시글을 클릭하면 미리보기 팝업이 뜹니다.
- 메인 미리보기 패널 삭제 및 게시판 흡수: 화면에 항상 떠 있던 독립
  미리보기 패널을 없애고, 게시판 글 클릭(또는 업로드 확인) 시 뜨는
  미리보기 팝업 하나로 통합했습니다. 팝업은 원형 X 버튼으로 닫습니다.
- 블로그 저장 한글화 및 저장 확인 팝업:
  - 표시명을 한글로 정리했습니다(Blogger 저장 → 블로그 저장 등).
  - 저장 시작 버튼을 눌러도 바로 저장하지 않고 "선택한 글을 Blogger
    임시글로 저장할까요?" 확인 팝업이 먼저 뜨며, [취소]/[저장하기] 중
    선택할 수 있습니다. [저장하기]를 눌러야 기존 저장 흐름이 실행되고,
    [취소]를 누르면 아무 작업 없이 팝업만 닫힙니다. 저장 함수 내부
    구현(runSaveFlow 등)은 v1.1과 동일합니다.
- 블로그 지시서 복사 모듈 추가(신규, 독립 기능):
  - 메인 화면에 [📋 블로그 지시서 복사] 버튼이 추가되었습니다.
  - 버튼을 누르면 팝업이 뜨고, [전체 복사]를 누르면
    data/blog-writing-prompt-v6.1.txt 원문 전체가 클립보드로 복사됩니다.
  - 클립보드 복사가 실패하는 환경에서는 textarea에 원문을 띄워 직접
    선택/복사할 수 있는 대체 수단을 제공합니다.
  - 이 기능은 글을 생성하지 않으며, GPT나 Gemini를 호출하지 않고,
    Blogger 저장과도 연결되어 있지 않습니다. 단순 텍스트 복사 기능입니다.
  - 주의: index.html을 file://로 직접 열면 브라우저 CORS 정책 때문에
    fetch()가 실패해 복사 팝업이 정상 동작하지 않을 수 있습니다.
    반드시 HTTP(S) 환경(로컬 서버, GitHub Pages 등)에서 실기 테스트가
    필요합니다.
- 팝업 공통 스타일 정리: 블로그 저장 확인/전체 게시판/미리보기/블로그
  지시서 복사 팝업 모두 기존 popup-panel 구조를 재사용합니다. 닫기
  버튼은 원형 안에 × 문자(aria-label="닫기")로 통일했고, 모바일에서
  터치하기 쉬운 크기로 조정했습니다. 팝업은 overlay: position: fixed;
  inset: 0; / panel: 화면 중앙 고정 / 팝업 내부만 스크롤되며, 드래그로
  이동하지 않습니다.
- v1.1 검수사항 반영(허용 파일 범위 안에서): Lite/영어 표시명 잔재를
  정리했고, Blogger 저장용 본문에서 table→카드 변환 시 새로 들어가던
  inline font-size(h3/p)를 제거했습니다(margin/line-height만 유지).

설치 방법

  을 그대로 GitHub Pages 저장소 루트에 덮어씁니다(상위 폴더를 한 번 더
  만들지 않습니다).
- v1.1 파일을 미리 백업해두는 것을 권장합니다.

실기 테스트 필요 항목
1. 로그인 → 통합 작업대 진입 확인
2. PC 브라우저에서 2열 패널 배치(업로드/블로그 저장 - 게시판), 모바일
   (또는 브라우저 폭 축소)에서 세로 배치 확인
3. 전체 페이지 스크롤 없이 각 패널/팝업 내부만 스크롤되는지 확인
4. ZIP 업로드 → 체크리스트 표시 확인 → 미리보기 팝업에서 본문 확인 →
   팝업 닫고 [게시판에 저장]
5. 게시판에서 글 클릭 → 미리보기 팝업이 뜨는지, [전체 게시판] 팝업에서도
   동일하게 동작하는지 확인
6. [블로그 임시저장 시작] 클릭 → 저장 확인 팝업 표시 → [취소] 클릭 시
   저장이 실행되지 않는지, [저장하기] 클릭 시 기존 저장 흐름이 정상
   실행되는지 확인
7. Blogger 임시저장 실행 → 표가 있던 자료가 Blogger 관리자 화면에서
   카드형으로 정상 표시되는지, 글씨 크기가 적절한지 확인
8. 이미지가 기존과 동일하게 정상 반영되는지 확인
9. [📋 블로그 지시서 복사] 버튼 → 팝업 → [전체 복사] 클릭 시 클립보드
   복사가 정상 동작하는지 확인(반드시 HTTP(S) 환경에서 테스트).
   클립보드 API를 사용할 수 없는 환경에서 textarea fallback이 뜨는지도
   함께 확인
