GPT 공작소 v1.8.6-fix2
게시판 확대 / 상태 배지 / 네이버 태그·이미지 보정

v1.8.6-fix1 실기(iPhone) 확인 후 나온 추가 보정입니다. 큰 구조 변경이
아니라 실기 기반 사용성 보정입니다. Blogger 임시저장/R2 업로드/Worker
API/로그인/ZIP 파싱 기본 로직은 전혀 건드리지 않았습니다.

- 게시판 확대: 게시판 영역 높이를 230px에서 290px로 늘리고, 업로드 박스는
  소폭 줄였습니다. 선택된 글 요약의 버튼도 다시 편안한 높이로 되돌려
  줄바꿈 없이 안정적으로 보입니다.
- 선정자료 단순화: TOP5 후보 섹션을 완전히 삭제했습니다(카테고리명만
  5번 반복 표시되어 정보 가치가 없었습니다). 이제 최종 TOP1과 선정
  이유만 보여줍니다.
- 상태 배지: 게시판/블로그스팟/네이버 헤더에 아이콘+색상 배지(🟦 선택됨,
  🟢 준비, ✅ 완료, 🔴 실패, 🟡 확인 필요)를 추가해 한눈에 상태를 볼 수
  있습니다. 기존 상태 판정 로직은 그대로 두고 표시만 보강했습니다.
- 네이버 태그 파싱 보정: naver_tags.txt 우선 사용은 그대로 두고, 공백
  1개만 있어도 개별 태그로 분리되도록 구분 규칙을 넓혔습니다(실기에서
  확인된 "긴 한 줄 태그" 문제 수정, "정보 없음" 같은 문구가 잘못
  쪼개지는 부작용도 함께 방지). 태그는 이제 쉼표로 이은 문자열이 아니라
  개별 칩(#태그)으로 표시되고, 복사 시에도 #을 붙입니다.
- 네이버 이미지 보정: 파일 다운로드 강제 방식을 없애고 "사진 저장용
  이미지 보기" 방식으로 바꿨습니다. [이미지 보기]를 누르면 중앙 고정
  팝업에 이미지가 화면 너비 안에서 크게 보이고, 길게 눌러 사진 앱에
  저장하도록만 안내합니다(자동 저장 시도·파일 앱 안내 없음).
- fallback 문구 정리: 업로드 대체 경로의 팝업 문구를 "업로드 완료/성공/
  실패/실패한 항목은 저장되지 않았습니다" 형태로 통일했습니다.
- 죽은 코드 정리: 더 이상 쓰이지 않는 CSS/함수를 실제로 제거했습니다.

실기 테스트 필요 항목
1. 로그인 화면 버전이 v1.6으로 보이는지 확인
2. 설정 → 버전 정보에서 현재 버전 v1.6 / 기준 버전 v1.5로 보이는지 확인
3. 상단 앱 헤더가 "🤖 GPT 공작소"로 보이는지 확인
4. 블로그 지시서 복사 버튼이 블로그 저장 카드 밖, 화면 맨 아래 독립
   영역에 있는지 확인
5. 잘못된 ZIP 업로드 후 설정 → 오류 목록에서 "[upload-module] 필수 파일
   누락" 대신 "블로그자료 ZIP 구조 오류"로 보이는지 확인(⚠️ 오류 목록
   팝업에서도 동일하게 확인)
6. 게시판에 글이 없을 때 안내 문구가 중앙에 정렬되고 잘리지 않는지 확인
7. 글을 1개 등록한 뒤 게시판 카드와 블로그 저장 카드(저장 버튼/최종
   미리보기/패키지 진단 버튼)가 모바일 폭에서 전부 잘리지 않고 보이는지 확인
8. 블로그 지시서 복사 팝업을 열고 [전체 복사]를 눌러도 팝업 높이가
   바뀌지 않는지 확인
9. 블로그 저장 확인 팝업이 내부 스크롤 없이 고정 크기로 보이는지 확인
10. Blogger 임시저장 진행 팝업이 로그가 늘어나도 크기가 고정되어 있는지
    확인(상단 요약 + 하단 고정 높이 로그)
11. 🩺 패키지 진단에서 최종 업데이트 날짜 위치/소제목 간격 항목이
    새로 표시되는지 확인
12. 🔍 최종 미리보기에서 발행 전 체크리스트 6항목과 모바일 프레임
    미리보기가 보이는지, 실제 저장/업로드 요청이 없는지(네트워크 탭)
    확인
13. 기본 업로드 → 게시판 저장 → R2 이미지 업로드 → Blogger 임시저장
    흐름이 이전과 동일하게 정상 동작하는지 확인
14. 신규 모듈 하나를 제거해도 기본 흐름이 유지되는지 확인(v1.4부터
    이어진 안전 연결 재검증)
