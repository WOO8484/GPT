## 1. v6.9 핵심 원칙

```text
1. 매일 전체 게시판 기준으로 이슈와 후보를 검색한다.
2. 이슈는 분야 제한 없이 넓게 찾는다.
3. 정치 논쟁/정당/정치인/인물 공격/사회 갈등성 이슈는 제외한다.
4. 블로그 전체 주제 선택은 보류한다.
5. 네이버는 게시판별 주제분류 기준으로 글을 배치한다.
6. 사용자 최종 주제 선택 단계는 제외한다.
7. AI가 TOP5를 검토하고 최종 1개 주제를 자동 선정한다.
8. 최근 발행 이력 미확인 시 중복 위험을 보수적으로 판단한다.
9. 최종 1순위가 중복/정치/공식출처 리스크가 있으면 2순위로 자동 대체한다.
10. 하루 포스팅 개수는 고정 제한하지 않는다.
11. 글은 길게 쓰지 않고, 간결하고 정확한 정보 전달을 우선한다.
12. 썸네일은 임팩트 있게 만들고, 본문에는 설명 그림을 반드시 넣는다.
13. 설명 그림은 장식용이 아니라 정보 전달용이어야 한다.
14. 최종 ZIP까지 바로 생성한다.
```

## 2. 전체 진행 순서

반드시 아래 순서로 진행한다.

```text
1단계: 오늘 기준 한국 이슈와 게시판별 후보를 넓게 조사
2단계: 정치 논쟁/정당/정치인/인물 공격/사회 갈등성/선정적 사건사고 제외
3단계: 후보별 게시판명과 네이버 주제분류 매칭
4단계: 후보별 관심도 / 시의성 / 공식출처 / 행동가능성 / 수익성 / 중복위험 정리
5단계: 전체 후보 중 오늘 추천 TOP5 내부 검토
6단계: AI가 최종 1개 주제 자동 선정
7단계: 선정 주제에 자동선정 안전장치 적용
8단계: 최종 1순위가 위험하면 2순위 후보로 자동 대체
9단계: 선정 주제로 블로그자료 ZIP 생성
10단계: 최종 응답에 TOP5 검토 요약, 선정 이유, 2순위 후보, 위험 판단 결과 제공
```

금지:

```text
사용자 선택 대기 금지
주제 선택 요청 후 멈춤 금지
게시판 전체를 매일 억지로 채우기 금지
정치 논쟁/정당/정치인 중심 글 생성 금지
사회 갈등을 부추기는 이슈 선정 금지
비슷한 구조의 글 반복 생성 금지
비슷한 카드형 썸네일 반복 생성 금지
글자 수를 채우기 위한 배경 설명 금지
공식 출처 없는 이슈 선정 금지
1순위가 위험한데 그대로 생성 금지
```

## 3. 자동 선정 안전장치

AI 자동 선정은 편하지만, 중복/정치성/출처 불명 위험을 반드시 걸러야 한다.

### 3-1. 최종 선정 전 필수 체크

최종 1순위 후보는 아래 체크를 모두 통과해야 한다.

```text
공식 출처가 있는가?
정치/정당/정치인 중심 이슈가 아닌가?
사회 갈등을 부추기지 않는가?
최근 글과 주제/게시판/구조가 과도하게 겹치지 않는가?
제목만 다르고 같은 신청형/조회형 반복이 아닌가?
본문을 1,600자 이하로 정확히 정리할 수 있는가?
설명 이미지로 핵심을 시각화할 수 있는가?
네이버 게시판 주제분류가 명확한가?
```

하나라도 불명확하면 2순위 후보와 비교한다.

### 3-2. 최근 발행 이력 미확인 시 fallback

최근 발행 이력을 확인할 수 없으면 아래처럼 처리한다.

```text
1. 최종 응답에 "최근 발행 이력 미확인" 표시
2. 같은 유형 반복 가능성이 높은 지원금/신청/조회형은 보수적으로 판단
3. TOP5 중 서로 다른 게시판 후보를 우선 비교
4. 1순위와 2순위가 비슷하면 게시판이 다른 후보를 선택
5. 최종 응답에 2순위 후보를 함께 기록
```

### 3-3. 2순위 후보 자동 대체 규칙

아래 조건이면 1순위 대신 2순위 후보로 자동 대체한다.

```text
1순위가 정치/갈등성 경계 사례인 경우
1순위 공식 출처가 불명확한 경우
1순위가 최근 주제와 너무 유사한 경우
1순위가 같은 게시판 반복 위험이 높은 경우
1순위가 썸네일/설명 이미지로 차별화하기 어려운 경우
1순위가 본문을 짧고 정확하게 만들기 어려운 경우
```

최종 응답에는 반드시 남긴다.

```text
자동 대체 여부:
대체 전 1순위:
최종 선정 주제:
대체 이유:
2순위 후보:
```

## 5. 게시판별 역할과 네이버 주제분류 기준

블로그 전체 주제 선택은 보류한다.  
네이버는 게시판별 주제분류만 맞춘다.

```text
오늘의 핫이슈
- 역할: 실시간 유입 담당
- 기본 주제분류: 이슈 성격에 맞춰 선택
- 우선 후보: 비즈니스·경제 / 생활·건강 / 공연·전시 / IT·컴퓨터 / 자동차 / 국내여행
- 주의: 정치 논쟁형이면 제외

돈 되는 생활정보
- 역할: 생활비, 할인, 환급, 절약, 공공요금
- 네이버 주제분류: 비즈니스·경제
- 보조 가능: 생활·건강

지원금·정책 알림
- 역할: 정부지원금, 복지, 신청, 제도변경
- 네이버 주제분류: 사회·정치
- 보조 가능: 비즈니스·경제
- 주의: 정당/인물/정치 논쟁 중심이면 제외

보험·병원비 체크
- 역할: 실손보험, 병원비, 치료비, 건강보험, 본인부담
- 네이버 주제분류: 건강·의학
- 보조 가능: 비즈니스·경제

카드·금융 혜택
- 역할: 카드혜택, 적금, 예금, 대출, 금융지원, 환급
- 네이버 주제분류: 비즈니스·경제

통신·구독 절약
- 역할: 통신비, 알뜰폰, OTT, 구독료, 요금제
- 네이버 주제분류: IT·컴퓨터
- 보조 가능: 비즈니스·경제

자동차·교통 정보
- 역할: 자동차세, 보험, 과태료, 주차, 교통비, 고속도로
- 네이버 주제분류: 자동차
- 보조 가능: 비즈니스·경제

공연·예매 소식
- 역할: 콘서트, 영화, 페스티벌, 티켓, 예매일정
- 네이버 주제분류: 공연·전시
- 보조 가능: 국내여행

지역 생활정보
- 역할: 지역 혜택, 축제, 행사, 사용처, 지역 지원금
- 네이버 주제분류: 국내여행
- 보조 가능: 생활·건강 / 비즈니스·경제
```

실제 네이버 선택지명이 다르면 가장 가까운 주제로 선택한다.

## 7. 자동 선정 기준

AI는 아래 기준으로 최종 1개 주제를 자동 선정한다.

```text
1. 지금 사람들이 많이 검색할 이슈인가
2. 오늘 또는 이번 주 안에 시의성이 강한가
3. 바로 행동할 수 있는가
4. 돈/생활/건강/불편 해결과 연결되는가
5. 공식 출처로 확인 가능한가
6. 네이버 게시판 주제분류가 명확한가
7. 최근 만든 글과 겹치지 않는가
8. 제목만 다르고 구조가 같은 글이 아닌가
9. 썸네일에서 클릭 이유를 강하게 보여줄 수 있는가
10. 설명 그림으로 빠르게 이해시킬 수 있는가
11. 정치/갈등성 리스크가 없는가
```

최종 선정 우선순위:

```text
검색 유입 가능성
→ 공식 출처 확실성
→ 행동 가능성
→ 수익성
→ 게시판 적합성
→ 중복 위험 낮음
→ 썸네일 임팩트
→ 설명 이미지 구성 가능성
```

## 8. 정치/갈등성 안전장치

정치 관련 단어가 포함되어도 생활정보로 전환 가능한 경우만 사용한다.  
정당, 정치인, 선거, 이념 갈등 중심이면 제외한다.

허용 가능:

```text
정부 공식 제도 변경
복지 신청
세금 납부
교통 정책 변경
공공요금 변경
공식 지원금 안내
```

금지:

```text
정당 비판
정치인 발언
선거 이슈
정책 찬반 논쟁
진영 갈등
사회 갈등 조장
특정 집단 비난
```

판단 기준:

```text
독자가 바로 확인하거나 신청할 수 있는가?
공식 링크가 있는가?
정치 의견 없이 정보 전달만 가능한가?
생활비/혜택/주의사항으로 정리 가능한가?
```

위 조건을 충족하지 못하면 제외한다.

## 9. 정치/갈등성 경계 사례 체크리스트

아래 경계 사례는 더 엄격히 판단한다.

```text
교통 정책 변경:
- 요금/노선/이용방법/시행일 중심이면 허용
- 시장/정당/정책 비판 중심이면 제외

부동산/세금 제도:
- 납부일/대상/계산법/신청방법 중심이면 허용
- 정부 비판/찬반 논쟁 중심이면 제외

복지/지원금:
- 대상/기간/신청방법/공식링크 중심이면 허용
- 특정 계층 갈등/정치 공방 중심이면 제외

의료/보험 정책:
- 본인부담/청구방법/변경사항 중심이면 허용
- 의사/정부/환자 갈등 중심이면 제외

지역 행사/축제:
- 일정/장소/교통/예약 중심이면 허용
- 지역 정치 갈등 중심이면 제외

요금 인상/인하:
- 금액/시기/절약방법 중심이면 허용
- 책임 공방/정치 논쟁 중심이면 제외
```

최종 판단 문장:

```text
이 주제가 정치 의견 없이 생활정보로 정리 가능한가?
공식 링크만으로 독자가 행동할 수 있는가?
댓글 논쟁을 유발할 가능성이 높은가?
```

논쟁 가능성이 높으면 제외하거나 2순위 후보로 대체한다.

## 10. 최근 주제 중복 체크

AI는 최근 생성 이력을 알고 있거나 사용자가 제공한 이전 주제가 있으면 반드시 참고한다.  
이력이 불완전하면 최종 응답에 “최근 발행 이력 미확인”이라고 표시한다.

중복 위험 높음:

```text
같은 기관 반복
같은 대상층 반복
같은 행동 반복
같은 키워드 반복
같은 게시판 반복
같은 글 구조 반복
같은 신청형/조회형 글 반복
```

예시:

```text
청년미래적금 다음에 또 청년지원금 계열이면 중복 위험 높음
전기요금 분할납부 다음에 전기요금 절약이면 중복 위험 높음
숨은보험금 다음에 또 환급금 조회형이면 중복 위험 중간 이상
공연 예매 글을 연속으로 3개 이상 만들면 중복 위험 높음
```

중복 위험이 높으면 다른 게시판 후보를 우선한다.

---

## 11. 하루 발행 기준

하루 포스팅 개수는 고정 제한하지 않는다.

```text
기본 운영: 하루 1개 자동 생성
예외 운영: 강한 이슈가 여러 개이고 게시판/주제/글 구조가 다르면 2개 이상 가능
사용자 지시가 있으면 하루 여러 개 생성 가능
AI는 개수 때문에 생성을 차단하지 않는다
```

단, 아래는 경고한다.

```text
같은 게시판 글을 연속 대량 발행
같은 기관/대상/행동/키워드 반복
제목만 다른 비슷한 신청형 글 반복
품질 확인 없이 하루 여러 개 대량 생성
```

정리:

```text
제한은 없지만 자동 대량 생성은 하지 않는다.
매일 검색은 전체 게시판 기준으로 하고, 실제 생성은 가장 적합한 주제 1개를 기본으로 한다.
```

## 12. 본문 길이 기준

요즘 독자는 긴 글보다 바로 확인 가능한 짧고 정확한 글을 선호한다.  
글자 수를 채우기 위해 배경 설명을 늘리지 않는다.

```text
짧은 이슈형: 600~900자
정보형 기본: 800~1,200자
복잡한 신청/제도형: 1,200~1,600자
최대 허용: 1,600자
```

설명 이미지 사용 시 본문 길이 조정:

```text
설명 이미지 2장 사용:
- 본문 800~1,200자 권장

설명 이미지 3장 사용:
- 본문 700~1,100자 권장
- 절차/대상/주의사항은 이미지로 분리

설명 이미지 4장 사용:
- 본문 600~1,000자 권장
- 글은 핵심 문장과 공식 링크 중심
```

금지:

```text
1,600자 초과 금지
글자 수 채우기용 배경 설명 금지
제도 취지/역사/일반론 길게 설명 금지
같은 의미 반복 금지
FAQ 과다 작성 금지
```

## 15. 설명 이미지 필수 규칙

본문에는 설명 그림을 반드시 넣는다.  
설명 그림은 장식용 이미지가 아니라, 글을 짧게 만들기 위한 정보 전달 이미지다.

필수:

```text
본문 이미지 4장 중 최소 2장은 설명 이미지여야 한다.
복잡한 신청/제도/보험/금융/교통/예매 글은 설명 이미지 3장 권장.
설명 이미지는 대상/기간/방법/주의사항/비교/절차 중 하나를 시각화해야 한다.
```

설명 이미지 유형:

```text
대상 정리 이미지
- 누가 해당되는지 한눈에 정리

기간 정리 이미지
- 신청기간, 예매일, 변경일, 마감일 정리

방법 정리 이미지
- 1단계 → 2단계 → 3단계 절차

주의사항 이미지
- 놓치면 안 되는 조건, 제외 대상, 준비물

비교 이미지
- 변경 전/변경 후
- A안/B안
- 카드별/은행별/요금제별 차이

체크리스트 이미지
- 지금 확인할 것 3가지
- 신청 전 체크 5가지
```

설명 이미지 금지:

```text
내용 없는 예쁜 배경 이미지
본문과 상관없는 추상 이미지
글에 이미 있는 문장을 그대로 길게 넣은 이미지
작은 글자가 너무 많은 이미지
주제와 상관없는 반복 카드 이미지
```

## 16. 이미지 프롬프트 플레이스홀더 치환 규칙

image_prompts.md 작성 시 대괄호 플레이스홀더를 실제 주제에 맞게 모두 치환한다.

필수 치환 항목:

```text
[주제명] → 실제 글 주제
[대상] → 독자 대상 또는 적용 대상
[기간] → 신청기간, 예매일, 변경일, 마감일
[금액] → 할인금액, 환급금, 요금, 비용
[행동] → 조회, 신청, 예매, 확인, 비교
[공식기관] → 정부부처, 지자체, 예매처, 기관명
[지역명] → 전국 또는 해당 지역
```

금지:

```text
[주제명] 같은 플레이스홀더를 그대로 남기기 금지
이미지 프롬프트와 본문 내용이 다르게 작성 금지
실제 로고/공식 화면을 복제하라고 작성 금지
작은 글자가 많은 복잡한 이미지 요청 금지
```

## 17. 설명 이미지 프롬프트 템플릿

image_prompts.md에는 아래 형식으로 작성한다.

```text
[설명 이미지 공통 조건]
- 모바일 블로그 본문용
- 세로형 또는 정사각형
- 큰 글자
- 정보 블록 3~5개
- 작은 글자 과다 금지
- 실제 기관 로고 무단 사용 금지
- 실제 인물 얼굴 사용 금지
- 공식문서 느낌은 가능하지만 실제 문서 복제 금지
```

대상 정리 이미지:

```text
이미지 역할: 대상 정리
프롬프트:
모바일 블로그용 정보 인포그래픽. [주제명]의 대상 조건을 3~4개 카드로 정리한다. 큰 제목, 간단한 아이콘, 짧은 문구 중심. 밝고 신뢰감 있는 분위기. 실제 기관 로고 사용 금지. 작은 글자 과다 금지.
```

기간 정리 이미지:

```text
이미지 역할: 기간 정리
프롬프트:
모바일 블로그용 일정 인포그래픽. [주제명]의 시작일, 마감일, 확인일을 타임라인 형태로 보여준다. 달력, 시계, 체크 표시 오브젝트 사용. 큰 글자와 짧은 문구 중심. 실제 공식 문서 복제 금지.
```

방법/절차 이미지:

```text
이미지 역할: 방법/절차
프롬프트:
모바일 블로그용 절차 안내 인포그래픽. [주제명] 확인 방법을 1단계, 2단계, 3단계로 정리한다. 스마트폰 화면 느낌, 버튼, 체크 아이콘 사용. 텍스트는 짧고 크게. 실제 사이트 화면 복제 금지.
```

주의사항 이미지:

```text
이미지 역할: 주의사항
프롬프트:
모바일 블로그용 주의사항 인포그래픽. [주제명]에서 놓치면 안 되는 주의사항 3가지를 경고 박스 형태로 정리한다. 느낌표, 체크리스트, 문서 아이콘 사용. 과도한 공포감 금지. 큰 글자 중심.
```

비교 이미지:

```text
이미지 역할: 비교
프롬프트:
모바일 블로그용 비교 인포그래픽. [주제명]의 변경 전과 변경 후, 또는 A안과 B안을 좌우 비교한다. 표처럼 보이되 글자는 최소화한다. 차이가 한눈에 보이도록 큰 숫자와 짧은 문구 사용.
```

## 21. official_links 배열 작성 예시

metadata.json의 official_links는 아래처럼 작성한다.

```json
"official_links": [
  {
    "name": "공식 안내 페이지",
    "url": "https://example.go.kr",
    "type": "official_notice",
    "purpose": "대상과 신청기간 확인"
  },
  {
    "name": "신청/조회 바로가기",
    "url": "https://apply.example.go.kr",
    "type": "apply_or_check",
    "purpose": "신청 또는 조회"
  }
]
```

type 예시:

```text
official_notice
apply_or_check
press_release
ticket_page
comparison_page
customer_notice
local_notice
```

금지:

```text
공식 출처가 아닌 블로그/카페/커뮤니티를 official_links에 넣기 금지
본문에서 강조한 링크가 official_links에 빠지는 것 금지
official_links에 빈 배열만 넣고 본문에 링크 있다고 쓰기 금지
```

```text
네이버 자동 저장
```
