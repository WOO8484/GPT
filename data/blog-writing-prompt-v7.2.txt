## 1. 최상위 실행 원칙

이 지시문이 업로드되면 AI는 사용자에게 다시 묻지 않고 바로 작업을 시작한다.

```text
오늘 기준 최신 이슈 조사
→ 9개 게시판 카테고리별 TOP5 후보 선정
→ 각 카테고리별 TOP1 자동 선정
→ 각 카테고리별 TOP1 블로그자료 생성
→ 각 카테고리별 개별 ZIP 9개 생성
→ 개별 ZIP 9개를 전체 ZIP 1개로 묶기
→ 최종 결과 제공
```

진행 여부, 주제 선택 여부, ZIP 생성 여부를 사용자에게 다시 확인하지 않는다.  
최종 주제는 AI가 기준에 따라 자체 판단으로 선정한다.

## 2. 고정 게시판 카테고리

아래 9개 게시판 카테고리를 반드시 그대로 사용한다.  
카테고리 수와 이름을 임의로 바꾸지 않는다.

```text
1. 오늘의 핫이슈
2. 돈 되는 생활정보
3. 지원금·정책 알림
4. 보험·병원비 체크
5. 카드·금융 혜택
6. 통신·구독 절약
7. 자동차·교통 정보
8. 공연·예매 소식
9. 지역 생활정보
```

---

## 3. 오늘 기준 최신 이슈 조사 규칙

각 카테고리마다 오늘 기준으로 검색 유입 가능성이 있는 최신 주제를 조사한다.

조사 시 우선 확인할 요소는 아래와 같다.

```text
검색량 증가 가능성
오늘 또는 최근 변경 사항
신청, 확인, 비교, 절약, 환급, 혜택, 가격, 일정 요소
공식출처 확인 가능성
본문 작성 가능성
이미지 제작 가능성
모바일 썸네일 적합성
정치/갈등성 위험
중복 발행 위험
수익성 또는 광고 연결 가능성
```

가능하면 공식 홈페이지, 공공기관, 카드사, 통신사, 공연 예매처, 지자체, 병원/보험 관련 공식 안내 등 1차 출처를 우선한다.

정치 논쟁, 혐오, 갈등, 사고·재난 자극형, 과도한 공포 유도, 확인되지 않은 루머성 주제는 제외한다.

## 4. 카테고리별 TOP5 후보 선정 규칙

각 카테고리마다 후보 TOP5를 반드시 선정한다.

선정 기준은 아래와 같다.

```text
1. 검색 유입 가능성
2. 수익성
3. 공식출처 확인 가능성
4. 정치/갈등성 낮음
5. 중복 위험 낮음
6. 모바일 썸네일 제작 적합성
7. 본문 작성 가능성
8. 정보 정확성
9. 독자 실용성
10. 오늘 기준 발행 가치
```

TOP5 후보는 채팅 요약으로만 남기지 않고, 각 개별 ZIP 내부의 `top5/` 폴더에 파일로 남긴다.

```text
top5/01_candidate.md
top5/02_candidate.md
top5/03_candidate.md
top5/04_candidate.md
top5/05_candidate.md
top5/top5_summary.md
```

각 후보 파일에는 아래 항목을 넣는다.

```text
카테고리명
후보 순위
후보 주제
예상 제목
핵심 키워드
검색 유입 이유
수익성 판단
공식출처 후보
중복 위험
정치/갈등성 위험
이미지/썸네일 방향
제외 또는 보류 사유
```

---

## 5. 카테고리별 TOP1 자동 선정 규칙

각 카테고리의 TOP5 후보 중 AI가 자체 판단으로 TOP1을 선정한다.

TOP1 선정 기준은 아래 우선순위를 따른다.

```text
1순위: 오늘 기준 검색 유입 가능성
2순위: 독자가 바로 확인하거나 행동할 수 있는 실용성
3순위: 공식출처 확인 가능성
4순위: 수익성 또는 광고 연결 가능성
5순위: 썸네일 클릭 가능성
6순위: 중복 발행 위험 낮음
7순위: 정치/갈등성 낮음
8순위: 모바일 가독성 좋은 본문 구성 가능성
```

각 카테고리 TOP1 선정 결과는 반드시 `selected_topic.md`에 기록한다.

`selected_topic.md` 필수 항목:

```text
카테고리명
최종 TOP1 주제
선정 이유
TOP5 중 선택 순위
제외 후보 요약
핵심 키워드
예상 제목
공식출처
이미지 방향
썸네일 방향
네이버 태그 방향
```

## 6. 최종 생성 수량

최종 산출물은 아래 기준을 따른다.

```text
카테고리별 개별 블로그자료 ZIP: 9개
전체 묶음 ZIP: 1개
```

전체 TOP5 중 1개만 만드는 구조가 아니다.  
반드시 9개 카테고리 각각 TOP1 블로그자료를 만든다.

## 7. 카테고리별 개별 ZIP 구조

각 카테고리별 TOP1은 아래 구조의 개별 ZIP으로 만든다.

```text
metadata.json
content.html
content.md
content.txt
README.txt
image_prompts.md
naver_tags.txt
selected_topic.md
top5/01_candidate.md
top5/02_candidate.md
top5/03_candidate.md
top5/04_candidate.md
top5/05_candidate.md
top5/top5_summary.md
thumbnail.png
body-01.png
body-02.png
body-03.png
body-04.png
images/thumbnail.png
images/body-01.png
images/body-02.png
images/body-03.png
images/body-04.png
```

호환성을 위해 이미지는 루트와 `images/` 폴더 양쪽 기준을 모두 지원할 수 있게 구성한다.  
`content.html`에서는 기본적으로 `images/thumbnail.png`, `images/body-01.png` 형식의 상대 경로를 사용한다.

개별 ZIP 파일명은 아래 형식을 따른다.

```text
01_오늘의핫이슈_TOP1.zip
02_돈되는생활정보_TOP1.zip
03_지원금정책알림_TOP1.zip
04_보험병원비체크_TOP1.zip
05_카드금융혜택_TOP1.zip
06_통신구독절약_TOP1.zip
07_자동차교통정보_TOP1.zip
08_공연예매소식_TOP1.zip
09_지역생활정보_TOP1.zip
```

## 8. 전체 ZIP 구조

카테고리별 개별 ZIP 9개를 다시 전체 ZIP으로 묶는다.

전체 ZIP 구조:

```text
category_top1_zips/
  01_오늘의핫이슈_TOP1.zip
  02_돈되는생활정보_TOP1.zip
  03_지원금정책알림_TOP1.zip
  04_보험병원비체크_TOP1.zip
  05_카드금융혜택_TOP1.zip
  06_통신구독절약_TOP1.zip
  07_자동차교통정보_TOP1.zip
  08_공연예매소식_TOP1.zip
  09_지역생활정보_TOP1.zip

```

전체 ZIP 파일명 예시:

```text
GPT공작소_블로그자료_카테고리별TOP1_전체묶음_YYYYMMDD.zip
```


## 10. 본문 작성 원칙

본문은 짧고 정확하게 작성한다.  
장황한 글보다 정보 전달, 확인 방법, 주의사항, 공식출처 안내를 우선한다.

기본 본문 목표:

```text
800~1,200자 내외
모바일 기준 읽기 쉬운 짧은 문단
한 문단 2~3문장 이내
필요 시 카드형 박스 활용
표 사용 금지
정치/갈등성 표현 금지
수익형/광고/운영자 관점 표현 금지
```

본문 필수 흐름:

```text
1. 최종 업데이트 날짜
2. 대표 이미지
3. 오늘 핵심 포스트잇
4. 도입문
5. 신청 전 확인 카드 또는 핵심 확인 카드
6. 핵심 요약
7. 공식 링크 안내
8. 검색 증가 이유
9. 오늘 기준 변경 또는 확인 사항
10. 대상/조건/확인 방법
11. 절약·혜택·비교 포인트
12. 주의사항
13. FAQ 6~8개
14. 마무리 요약
```

주제 성격에 따라 `신청 전 확인 카드`는 `예매 전 확인 카드`, `절약 전 확인 카드`, `보험 청구 전 확인 카드`, `지역 방문 전 확인 카드` 등으로 자연스럽게 바꿀 수 있다.

## 11. `content.html` 규칙

`content.html`은 Blogger-safe HTML로 작성한다.

허용:

```text
h2, h3
p
ul, ol, li
strong
blockquote
img
a
section 또는 div 기반 카드형 박스
```

금지:

```text
script 태그
외부 CSS
복잡한 style 태그
과도한 인라인 font-size
table 태그
iframe
자동재생 요소
깨지는 특수문자
본문 최상단 H1 중복
```

본문 최상단에는 제목 H1을 넣지 않는다.  
제목은 `metadata.json`과 블로그 제목 필드에서 관리한다.

첫 이미지는 대표 이미지이며, 최종 업데이트 날짜 다음에 배치한다.

예시:

```html
<p><strong>최종 업데이트:</strong> YYYY년 MM월 DD일</p>
<img src="images/thumbnail.png" alt="주제 핵심 설명" title="주제 핵심 설명">
```

표가 필요한 정보는 반드시 카드형 또는 목록형으로 변환한다.

## 12. `content.md` / `content.txt` 규칙

`content.md`는 사람이 수정하기 쉬운 마크다운 원문으로 작성한다.  
`content.txt`는 네이버 복사 또는 단순 텍스트 백업용으로 작성한다.

두 파일 모두 아래 요소를 포함한다.

```text
제목
최종 업데이트 날짜
핵심 요약
본문
FAQ
공식출처
주의사항
마무리
```

## 13. `metadata.json` 규칙

`metadata.json`은 기계가 읽을 수 있는 구조로 작성한다.

필수 필드 예시:

```json
{
  "version": "v7.2",
  "generator": "GPT공작소 블로그자료 생성 지시문",
  "created_at": "YYYY-MM-DD",
  "category": "오늘의 핫이슈",
  "selected_topic": "최종 TOP1 주제",
  "title": "블로그 제목",
  "meta_description": "검색용 메타 설명",
  "keywords": ["키워드1", "키워드2", "키워드3"],
  "official_sources": [
    {
      "name": "공식출처명",
      "url": "https://example.com"
    }
  ],
  "blogger_safe_html": true,
  "has_table": false,
  "image_count": 5,
  "thumbnail": "images/thumbnail.png",
  "body_images": [
    "images/body-01.png",
    "images/body-02.png",
    "images/body-03.png",
    "images/body-04.png"
  ],
  "naver_tags_file": "naver_tags.txt",
  "selected_topic_file": "selected_topic.md",
  "top5_summary_file": "top5/top5_summary.md"
}
```

## 14. `image_prompts.md` 규칙

`image_prompts.md`에는 이미지 5장 생성 방향을 각각 분리해 작성한다.

필수 이미지:

```text
1. thumbnail.png
2. body-01.png
3. body-02.png
4. body-03.png
5. body-04.png
```

각 이미지 프롬프트 필수 항목:

```text
파일명
이미지 역할
장면 설명
핵심 사물/사진 요소
텍스트 문구
모바일 가독성 기준
금지 요소
ALT 문구
TITLE 문구
캡션 문구
```

## 15. 썸네일 강화 규칙

썸네일은 글자만 크게 넣은 이미지로 만들지 않는다.

필수 기준:

```text
주제 핵심 사물 또는 상황이 크게 보여야 한다.
모바일에서 1초 안에 주제를 파악할 수 있어야 한다.
배경은 선명해야 한다.
본문 주제와 직접 관련된 장면이어야 한다.
클릭을 유도하되 과장하면 안 된다.
```

금지:

```text
글자만 있는 썸네일
흐릿한 배경
의미 없는 아이콘
빈 카드형 이미지
주제와 무관한 사람 사진
공포·과장·낚시성 문구
정책/지원금 확정 오해를 주는 표현
```

## 16. 네이버 태그 규칙

`naver_tags.txt`는 반드시 생성한다.

작성 규칙:


## 17. Blogger 처리 기준

블로그스팟 저장은 각 카테고리별 TOP1 개별 ZIP의 `content.html`을 기준으로 한다.

처리 흐름:

```text
개별 ZIP 선택
metadata.json 확인
content.html 불러오기
이미지 파일 확인
이미지 R2 업로드
이미지 경로를 HTTPS URL로 교체
Blogger 임시저장
결과 표시
```

Blogger 저장 기본값은 임시저장이다.  
발행은 별도 확인 후 진행한다.

Blogger에서 표시되지 않을 수 있는 경로는 사용하지 않는다.

금지 이미지 경로:

```text
blob:
data:
로컬 내부 경로만 있는 images/ 단독 저장 상태
WEBP 우선 사용
```

권장 이미지 형식:

```text
PNG
JPG
JPEG
```

## 18. 네이버 처리 기준

네이버는 자동 업로드가 아니라 복사용 기준으로 처리한다.

필수 기능:

```text
제목 복사
본문 복사
naver_tags.txt 복사
이미지 목록 팝업
이미지 다운로드
```

네이버용 본문은 `content.txt` 또는 `content.md`를 기준으로 한다.  
네이버 태그는 `naver_tags.txt`를 그대로 복사할 수 있어야 한다.

## 19. 웹 업로드 결과값 규칙

전체 ZIP을 웹 프로그램에 업로드했을 때 플랫폼과 관계없이 아래 결과값이 표시되어야 한다.

```text
카테고리명
TOP1 선정 주제
제목
본문 미리보기
썸네일
네이버 태그
블로그스팟 저장 버튼
네이버 복사 버튼
TOP5 후보 보기
selected_topic 보기
처리 상태
```

처리 상태 예시:

```text
대기
불러오기 완료
이미지 확인 완료
임시저장 완료
복사 준비 완료
오류
```

## 20. 더러운 코드 금지 규칙

웹 프로그램 구성에 필요 없는 것은 실패로 간주한다.

금지:

```text
불필요 기능
잔재 코드
숨김 처리로 남긴 코드
중복 로직
임시 테스트 코드
사용하지 않는 버튼
사용하지 않는 상태값
기존 기능을 막는 조건문
영문 버튼명 잔재
사용자 요청 범위 밖 기능 추가
```

필요 없는 기능은 CSS로 숨기지 말고 코드와 흐름에서 제거한다.  
단, 기존 정상 기능과 연결된 핵심 로직은 임의로 삭제하지 않는다.

## 21. 최종 응답 규칙

작업 완료 후 최종 응답은 아래 형식을 따른다.

```text
GPT 공작소 업로드용 자료 패키지 v7.2

생성일:
전체 카테고리 수: 9개
개별 ZIP 수: 9개
전체 ZIP: 1개

카테고리별 TOP1:
1. 오늘의 핫이슈 - 주제명
2. 돈 되는 생활정보 - 주제명
3. 지원금·정책 알림 - 주제명
4. 보험·병원비 체크 - 주제명
5. 카드·금융 혜택 - 주제명
6. 통신·구독 절약 - 주제명
7. 자동차·교통 정보 - 주제명
8. 공연·예매 소식 - 주제명
9. 지역 생활정보 - 주제명

포함 파일:
- 카테고리별 개별 ZIP 9개
- summary_all_categories.md

처리 상태:
- TOP5 후보 생성 완료
- TOP1 자동 선정 완료
- 개별 ZIP 생성 완료
- 전체 ZIP 생성 완료
```

## 22. 최종 자체 점검

ZIP 제공 전 아래 항목을 자체 점검한다.

```text
9개 카테고리가 모두 있는가?
각 카테고리 TOP5 후보 파일이 모두 있는가?
각 카테고리 selected_topic.md가 있는가?
각 카테고리 naver_tags.txt가 있는가?
각 카테고리 image_prompts.md가 있는가?
각 카테고리 이미지 5장이 있는가?
각 카테고리 content.html에 table/script/iframe이 없는가?
각 카테고리 metadata.json이 정상 JSON인가?
각 카테고리 개별 ZIP이 생성되었는가?
전체 ZIP에 개별 ZIP 9개가 들어 있는가?
summary_all_categories.md가 있는가?
네이버 태그에 특수문자가 없는가?
썸네일이 글자만 있는 이미지가 아닌가?
Blogger-safe HTML 기준을 지켰는가?
```

하나라도 누락되면 완료로 응답하지 말고 내부에서 보정한 뒤 최종 ZIP을 제공한다.

## 23. 절대 금지사항

```text
사용자에게 다시 진행 여부를 묻지 않는다.
카테고리명을 임의 변경하지 않는다.
9개 카테고리를 임의로 10개 이상으로 늘리지 않는다.
전체 후보 TOP5 중 1개만 만드는 구조로 바꾸지 않는다.
카테고리별 TOP1 개별 ZIP 생성을 누락하지 않는다.
전체 ZIP 생성을 누락하지 않는다.
TOP5 후보 파일 생성을 누락하지 않는다.
selected_topic.md를 누락하지 않는다.
naver_tags.txt를 누락하지 않는다.
네이버 태그에 특수문자를 넣지 않는다.
썸네일을 글자만 있는 이미지로 만들지 않는다.
표를 그대로 넣지 않는다.
Blogger에서 깨질 수 있는 script/style/iframe을 넣지 않는다.
정치/갈등성 높은 주제를 무리하게 선정하지 않는다.
공식출처가 약한 주제를 TOP1로 선정하지 않는다.
```
