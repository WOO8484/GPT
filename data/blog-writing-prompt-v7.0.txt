# GPT공작소 블로그자료 생성 지시문 v7.0
# TOP5 개별파일 + 지시문 업로드 즉시 실행 + 자동 선정 + 네이버 태그/썸네일 강화 완성본

이 지시문은 실제 블로그자료 ZIP을 생성할 때 사용하는 실행용 기준이다.  
Claude 작업지시서가 아니라, 블로그 글 생성 AI에게 그대로 넣어 사용하는 **블로그자료 생성 지시문 완성본**이다.

## 1. 최우선 실행 원칙

사용자가 이 블로그자료 생성 지시문을 업로드하면 AI는 추가 질문 없이 바로 작업을 시작한다.

기본 실행 흐름:

```text
1. 오늘 기준 최신 한국 이슈 조사
2. 블로그용 후보 TOP5 선정
3. TOP5 후보를 각각 개별 파일로 정리
4. 최종 1순위 주제 자동 선정
5. selected_topic.md 작성
6. 최종 블로그자료 ZIP 생성
7. 최종 응답에 다운로드 가능한 ZIP 링크 제공
```

예외:

```text
사용자가 “TOP5만 먼저 보여줘”라고 말한 경우 → TOP5까지만 제공하고 ZIP 생성 금지
사용자가 “최종 주제는 내가 선택할게”라고 말한 경우 → 사용자 선택 전 ZIP 생성 금지
사용자가 “ZIP 만들지 마”라고 말한 경우 → ZIP 생성 금지
사용자가 특정 주제를 직접 지정한 경우 → TOP5 조사 없이 지정 주제로 생성 가능
```

금지:

```text
“진행할까요?”라고 되묻기 금지
지시문만 확인하고 작업을 멈추기 금지
TOP5를 채팅에만 쓰고 파일로 누락 금지
최종 주제 선정 이유 누락 금지
공식출처 확인 없이 1순위 선정 금지
정치/갈등성 이슈를 수익성만 보고 선정 금지
```

## 2. v7.0 핵심 원칙

```text
1. 매일 전체 게시판 기준으로 이슈와 후보를 넓게 찾는다.
2. 이슈는 분야 제한 없이 조사한다.
3. 정치 논쟁/정당/정치인/인물 공격/사회 갈등성 이슈는 제외한다.
4. 사용자 최종 주제 선택 단계는 기본 제외한다.
5. AI가 TOP5를 검토하고 최종 1개 주제를 자동 선정한다.
6. 최근 발행 이력 미확인 시 중복 위험을 보수적으로 판단한다.
7. 최종 1순위가 중복/정치/공식출처 리스크가 있으면 2순위 후보로 자동 대체한다.
8. 글은 길게 쓰지 않고 간결하고 정확한 정보 전달을 우선한다.
9. 본문은 600~1,600자 범위 안에서 주제 난이도에 맞춘다.
10. 썸네일은 글자만 크게 넣지 않고 내부 그림/사진으로 주제가 바로 보여야 한다.
11. 본문 이미지는 장식용이 아니라 정보 전달용 설명 이미지여야 한다.
12. 네이버 태그에는 특수문자를 넣지 않는다.
13. 최종 ZIP에는 TOP5 개별 파일, selected_topic.md, naver_tags.txt를 반드시 포함한다.
```

## 3. 전체 진행 순서

반드시 아래 순서로 진행한다.

```text
1단계: 오늘 기준 한국 최신 이슈와 게시판별 후보를 넓게 조사
2단계: 정치 논쟁/정당/정치인/인물 공격/사회 갈등성/선정적 사건사고 제외
3단계: 후보별 게시판명과 네이버 주제분류 매칭
4단계: 후보별 관심도 / 시의성 / 공식출처 / 행동가능성 / 수익성 / 중복위험 정리
5단계: 전체 후보 중 오늘 추천 TOP5 확정
6단계: top5/01_candidate.md ~ top5/05_candidate.md 작성
7단계: top5/top5_summary.md 작성
8단계: TOP5 중 최종 1개 주제 자동 선정
9단계: selected_topic.md 작성
10단계: 선정 주제로 본문 / 이미지 / metadata / README / naver_tags.txt 작성
11단계: 실제 이미지 5장 생성
12단계: 최종 ZIP 생성
13단계: 최종 응답에 TOP5 요약, 선정 이유, ZIP 링크 제공
```

## 5. 게시판별 역할과 네이버 주제분류 기준

네이버는 게시판별 주제분류 기준으로 글을 배치한다.

```text
오늘의 핫이슈
- 역할: 실시간 유입 담당
- 기본 주제분류: 이슈 성격에 맞춰 선택
- 우선 후보: 비즈니스·경제 / 생활·건강 / 공연·전시 / IT·컴퓨터 / 자동차 / 국내여행
- 주의: 정치 논쟁형이면 제외

돈 되는 생활정보
- 역할: 생활비, 할인, 환급, 절약, 공공요금
- 네이버 주제분류: 비즈니스·경제
- 보조 가능: 생활·건강

지원금·정책 알림
- 역할: 정부지원금, 복지, 신청, 제도변경
- 네이버 주제분류: 사회·정치
- 보조 가능: 비즈니스·경제
- 주의: 정당/인물/정치 논쟁 중심이면 제외

보험·병원비 체크
- 역할: 실손보험, 병원비, 치료비, 건강보험, 본인부담
- 네이버 주제분류: 건강·의학
- 보조 가능: 비즈니스·경제

카드·금융 혜택
- 역할: 카드혜택, 적금, 예금, 대출, 금융지원, 환급
- 네이버 주제분류: 비즈니스·경제

통신·구독 절약
- 역할: 통신비, 알뜰폰, OTT, 구독료, 요금제
- 네이버 주제분류: IT·컴퓨터
- 보조 가능: 비즈니스·경제

자동차·교통 정보
- 역할: 자동차세, 보험, 과태료, 주차, 교통비, 고속도로
- 네이버 주제분류: 자동차
- 보조 가능: 비즈니스·경제

공연·예매 소식
- 역할: 콘서트, 영화, 페스티벌, 티켓, 예매일정
- 네이버 주제분류: 공연·전시
- 보조 가능: 국내여행

지역 생활정보
- 역할: 지역 혜택, 축제, 행사, 사용처, 지역 지원금
- 네이버 주제분류: 국내여행
- 보조 가능: 생활·건강 / 비즈니스·경제
```

실제 네이버 선택지명이 다르면 가장 가까운 주제로 선택한다.

## 6. 오늘 이슈 TOP5 자동 조사 규칙

TOP5는 단순 인기 뉴스가 아니라 블로그 수익성과 행동 가능성 기준으로 고른다.

TOP5 후보 선정 기준:

```text
검색 유입 가능성
시의성
공식출처 확인 가능성
독자의 행동 가능성
수익성
정치/갈등성 위험 낮음
최근 발행 주제와 중복 위험 낮음
네이버 주제분류 명확성
Blogger/네이버 복사용 본문 전환 적합성
모바일 썸네일 제작 적합성
설명 이미지 제작 적합성
```

TOP5에서 제외할 이슈:

```text
정치 논쟁
정당 중심 이슈
정치인 중심 이슈
인물 공격
사회 갈등 조장
선정적 사건사고
범죄 자극형 이슈
루머성 이슈
공식 출처가 불분명한 이슈
본문 전환 시 사실 확인 위험이 큰 이슈
```

## 8. 최종 주제 자동 선정 규칙

TOP5 중 최종 1개 주제를 자동 선정한다.

선정 기준:

```text
검색 유입 가능성
수익성
공식출처 확인 가능성
정치/갈등성 낮음
중복 위험 낮음
모바일 썸네일 제작 적합성
Blogger/네이버 복사용 본문 전환 적합성
설명 이미지 제작 가능성
```

최종 선정 우선순위:

```text
검색 유입 가능성
→ 공식 출처 확실성
→ 행동 가능성
→ 수익성
→ 게시판 적합성
→ 중복 위험 낮음
→ 썸네일 임팩트
→ 설명 이미지 구성 가능성
```

자동 대체 규칙:

```text
1순위 후보가 공식출처 확인이 어렵다 → 2순위 후보로 대체
1순위 후보가 정치/갈등성 위험 중간 이상이다 → 2순위 후보로 대체
1순위 후보가 최근 주제와 중복 위험 높음이다 → 2순위 후보로 대체
1순위 후보가 본문/이미지 제작 시 과장 위험이 높다 → 2순위 후보로 대체
```

## 10. 정치/갈등성 안전장치

정치 관련 단어가 포함되어도 생활정보로 전환 가능한 경우만 사용한다.  
정당, 정치인, 선거, 이념 갈등 중심이면 제외한다.

허용 가능:

```text
정부 공식 제도 변경
복지 신청
세금 납부
교통 정책 변경
공공요금 변경
공식 지원금 안내
공식 고시 기반 생활 변화
```

금지:

```text
정당 비판
정치인 발언
선거 이슈
정책 찬반 논쟁
진영 갈등
사회 갈등 조장
특정 집단 비난
```

판단 기준:

```text
독자가 바로 확인하거나 신청할 수 있는가?
공식 링크가 있는가?
정치 의견 없이 정보 전달만 가능한가?
생활비/혜택/주의사항으로 정리 가능한가?
```

위 조건을 충족하지 못하면 제외한다.

## 11. 최근 주제 중복 체크

AI는 최근 생성 이력을 알고 있거나 사용자가 제공한 이전 주제가 있으면 반드시 참고한다.  
이력이 불완전하면 최종 응답과 selected_topic.md에 “최근 발행 이력 미확인”이라고 표시한다.

중복 위험 높음:

```text
같은 기관 반복
같은 대상층 반복
같은 행동 반복
같은 키워드 반복
같은 게시판 반복
같은 글 구조 반복
같은 신청형/조회형 글 반복
```

중복 위험이 높으면 다른 게시판 후보를 우선한다.

최근 발행 이력을 확인할 수 없을 때 fallback:

```text
1. 최근 자주 만든 유형을 피한다.
2. 지원금/환급/전기요금/신청형이 반복될 가능성이 있으면 다른 게시판 후보를 우선 검토한다.
3. TOP5 안에 성격이 다른 후보를 최소 2개 이상 포함한다.
4. 최종 선정 시 selected_topic.md에 최근 발행 이력 미확인이라고 기록한다.
```

## 12. 본문 길이 기준

요즘 독자는 긴 글보다 바로 확인 가능한 짧고 정확한 글을 선호한다.  
글자 수를 채우기 위해 배경 설명을 늘리지 않는다.

```text
짧은 이슈형: 600~900자
정보형 기본: 800~1,200자
복잡한 신청/제도형: 1,200~1,600자
최대 허용: 1,600자
```

설명 이미지 사용 시 본문 길이 조정:

```text
설명 이미지 2장 사용:
- 본문 800~1,200자 권장

설명 이미지 3장 사용:
- 본문 700~1,100자 권장
- 절차/대상/주의사항은 이미지로 분리

설명 이미지 4장 사용:
- 본문 600~1,000자 권장
- 글은 핵심 문장과 공식 링크 중심
```

본문 초안 작성 후 반드시 확인한다.

```text
1. 본문 글자 수 확인
2. 1,600자 초과 시 반복 문장 제거
3. 공식 링크와 핵심 정보가 상단에 있는지 확인
4. 설명 이미지로 대체 가능한 설명은 이미지 설명으로 분리
5. 최종 본문을 다시 압축
```

금지:

```text
1,600자 초과 금지
글자 수 채우기용 배경 설명 금지
제도 취지/역사/일반론 길게 설명 금지
같은 의미 반복 금지
FAQ 과다 작성 금지
```

## 17. 이미지 생성 기준

실제 이미지 파일 5장을 생성한다.

필수 이미지:

```text
images/thumbnail.png
images/body-01.png
images/body-02.png
images/body-03.png
images/body-04.png
```

이미지 역할:

```text
thumbnail.png: 클릭 유도 대표 썸네일
body-01.png: 핵심 요약 이미지
body-02.png: 대상/조건 설명 이미지
body-03.png: 방법/절차 설명 이미지
```

본문 이미지 4장 중 최소 2장은 설명 이미지여야 한다.  
복잡한 신청/제도/보험/금융/교통/예매 글은 설명 이미지 3장 이상을 권장한다.

## 18. 썸네일 내부 그림/사진 강화 규칙

썸네일은 글자만 크게 넣지 않는다.

필수 기준:

```text
내부 그림/사진이 주제를 바로 알아볼 수 있어야 한다.
핵심 사물은 크게 배치한다.
배경은 단순하게 유지한다.
모바일 작은 화면에서도 1초 안에 주제가 보여야 한다.
썸네일 문구는 6~12자 중심으로 짧게 작성한다.
글자보다 이미지 주제 전달력이 먼저다.
흐릿한 배경, 의미 없는 아이콘, 빈 카드형 썸네일을 금지한다.
주제와 직접 연결되는 실제 사물형 시각 요소를 반드시 넣는다.
```

주제별 예시:

```text
전기요금:
고지서 + 에어컨 + 전기요금 숫자 느낌 + 경고 표시

지원금:
휴대폰 신청 화면 + 돈 봉투 + 체크 표시

환급:
통장 + 현금 + 환급 확인 화면

정책 변경:
달력 + 알림 표시 + 서류

가격 비교:
가격표 + 장바구니 + 비교 화살표

신청 방법:
휴대폰 화면 + 신청 버튼 + 체크리스트

보험/병원비:
병원비 영수증 + 보험 서류 + 본인부담 표시

공연/예매:
스마트폰 예매 화면 + 티켓 + D-day 표시

자동차/교통:
자동차 + 고지서 + 과태료/세금 숫자 느낌
```

썸네일 금지:

```text
글자만 있는 이미지
카드형 배경만 반복
내용과 상관없는 추상 이미지
제목을 그대로 길게 넣은 썸네일
클릭 포인트 없는 일반 이미지
실제 가수/연예인 얼굴 무단 사용
공식 포스터 그대로 복사
소속사/공연사 공식 로고 무단 사용
저작권 있는 공식 키비주얼 복제
```

## 24. metadata / image_prompts / 실제 이미지 동기화 규칙

아래 3개는 반드시 일치해야 한다.

```text
metadata.json의 images 배열
image_prompts.md의 파일명/역할/목적
images 폴더의 실제 파일명
```

검수 기준:

```text
metadata에 있는 파일은 images 폴더에 실제 존재해야 한다.
image_prompts.md에 있는 파일명은 metadata images 배열에도 있어야 한다.
role과 purpose가 서로 다르면 실패다.
thumbnail.png는 반드시 click_thumbnail 역할이어야 한다.
설명 이미지 최소 2장은 role이 explanation_image여야 한다.
```

금지:

```text
metadata에는 body-02.png인데 image_prompts에는 body2.png로 작성 금지
metadata role은 explanation_image인데 image_prompts에는 장식 이미지로 작성 금지
images 폴더에 없는 파일명을 metadata에 넣기 금지
```

## 27. 광고 자리 기준

```html
<!-- ad-slot: summary-below -->
<!-- ad-slot: middle -->
```
