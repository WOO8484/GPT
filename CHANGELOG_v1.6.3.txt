3. 추가 기능
------------------------------------------------------------
- 상단 헤더 지시서 복사 아이콘: 📋(지시서 복사) ⚠️(오류 목록) ⚙️(설정)

  동일합니다.
- 블로그스팟 카드 명칭 명확화: "블로그 저장"→"블로그스팟".
- 네이버 카드 신규 추가(js/naver-export-module.js):
  · getNaverTitle / getNaverBody / getNaverTags (읽기 전용 함수)
  · copyTitle / copyBody / copyTags (클립보드 복사 + 카드 내 안내 문구)
  · renderNaverImageList (이미지 순서 목록 표시, 자동 삽입 없음)
  · 제목 복사 우선순위: 선택된 글 title → metadata.title → 본문 첫
    제목(h1/h2/h3) → 파일명 기반 제목
  · 본문 복사 우선순위: content.txt → content.md(마크다운 기호 정리) →
    content.html(태그 제거 후 텍스트 변환, 이미지 위치는
    "[사진 N 삽입 위치]"로 표시). 본문 하단에 해시태그를 기본 포함.
  · 태그 복사 우선순위: metadata.tags → metadata.keywords → 본문 하단
    #태그 추출. "#태그1 #태그2" 형식으로 복사.
  · 네이버 자동 로그인/자동 발행/네이버 앱 제어/에디터 자동 입력은
    1차 범위가 아니므로 구현하지 않았습니다.

참고: 남은 주의사항
------------------------------------------------------------
- 이번 작업은 index.html의 카드 배치/문구, css/layout.css·
  css/components.css의 "추가" 규칙, js/naver-export-module.js 신규
  파일로만 한정했습니다. 요청하신 2가지 범위(메인페이지 재구성, 네이버
  모듈 추가) 외의 리팩토링/정리/삭제는 진행하지 않았습니다.
- 네이버 카드는 읽기 전용 보조 기능입니다. 문제가 발생하면
  index.html에서 <section class="panel panel--naver">...</section>
  블록과 <script src="js/naver-export-module.js"> 태그, 그리고
  css/layout.css·css/components.css 끝부분의 네이버 전용 규칙만
  제거하면 기존 시스템(로그인/업로드/게시판/블로그스팟 저장)은 100%
  그대로 복구됩니다.
- 실제 서비스 배포 전, 위 "코드 리뷰로 확인" 항목(실기 화면, 실제
  Worker 서버 연동)은 사용자 환경에서 직접 최종 확인을 권장합니다.
