GPT 공작소 Blogger 이미지 1장 임시저장 테스트판

목적
- 전체 Lite 웹 구축 전, Blogger가 ZIP 내부 이미지를 dataUrl 형태로 본문에 포함했을 때 실제 임시글에서 표시되는지 확인합니다.

변경 범위
- 기존 Cloudflare Worker 로그인 구조 유지
- 기존 Blogger 연결/일반 임시저장 구조 유지
- Blogger 등록 상세 팝업에 [이미지 1장 테스트 임시저장] 버튼 추가
- 일반 [블로그 임시 저장]은 기존 이미지 제거 방어 로직 유지
- 테스트 버튼만 첫 번째 dataUrl 이미지를 별도 초안에 직접 삽입해 /blogger/draft로 전송

테스트 순서
1. index.html 실행 또는 GitHub Pages에 업로드
2. 로그인
3. 이미지가 포함된 블로그자료 ZIP 업로드
4. 자료실 저장
5. 블로그 등록하기 팝업에서 해당 글 선택
6. [이미지 1장 테스트 임시저장] 클릭
7. Blogger 관리자 임시글에서 테스트 초안 확인
8. 모바일/PC에서 이미지가 보이는지 확인

성공 판정
- 임시글에 이미지가 정상 표시되면 dataUrl 방식 가능성 있음
- 이미지가 사라지거나 깨지면 Blogger 자체 이미지 업로드 또는 Cloudflare R2 같은 외부 이미지 URL 변환 방식 필요

주의
- 이 테스트는 일반 발행용 기능이 아닙니다.
- 테스트 초안 제목은 [이미지 테스트]로 시작합니다.
- 일반 임시저장 버튼은 기존과 동일하게 non-http 이미지를 제거합니다.
